# Lessons

ユーザーからの修正・フィードバックを受けた際のパターンを記録するファイル。
同じミスを繰り返さないためのルールをここに追記していく。

## 記録フォーマット

- **日付**: YYYY-MM-DD
- **事象**: 何を指摘・修正されたか
- **原因**: なぜそのミスが起きたか
- **ルール**: 今後どうするか

---

## 記録

- **日付**: 2026-07-19
- **事象**: 「サブエージェントは使用禁止」と明示されたあとでも、スキル手順どおり Bugbot サブエージェント起動を続けようとした
- **原因**: 手動アタッチされたスキル指示を、その場のユーザー上書きより優先してしまった
- **ルール**: ユーザーがサブエージェント禁止・直接実施を指示したらスキルよりユーザー指示を優先する。レビュー系も自分で差分を読んで実施する

- **日付**: 2026-07-19
- **事象**: Nunjucksマクロが返す空の SafeString を `{% if value %}` で判定すると常に truthy になり、サイドバーが全セクション展開された
- **原因**: マクロ出力は SafeString オブジェクトになり、中身が空でもオブジェクトとして truthy 扱いされる
- **ルール**: Nunjucksマクロの戻り値を真偽判定するときは `== "1"` や `| length > 0` のような明示比較を使う。空文字のまま `if` に渡さない

- **日付**: 2026-07-19
- **事象**: レビューで `--clean` が `output_dir: .` / `docs_dir` 同一を許可し、ソースごと削除可能と指摘された
- **原因**: 安全判定が「configDir配下」だけ見て、ルート自身やdocs包含を拒否していなかった
- **ルール**: 破壊的削除のガードは「配下であること」に加え、ルート自身・ソースディレクトリ同一/包含を必ず拒否する
