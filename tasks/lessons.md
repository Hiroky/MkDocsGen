# Lessons

ユーザーからの修正・フィードバックを受けた際のパターンを記録するファイル。
同じミスを繰り返さないためのルールをここに追記していく。

## 記録フォーマット

- **日付**: YYYY-MM-DD
- **事象**: 何を指摘・修正されたか
- **原因**: なぜそのミスが起きたか
- **ルール**: 今後どうするか

- **原因**: markdown-itのhrefエンコードとslugifyの生文字列をそのまま比較していた

- **事象**: 静的サイトなのに検索インデックスを fetch 前提にしており、file:// 直接閲覧を想定していなかった
- **原因**: ローカルHTTPサーバーでの確認を暗黙の前提にしてしまった
- **ルール**: 出力はスタンドアロン静的ページとして file:// で完結させる。データ読込は fetch ではなく相対パスの script/link を使う

- **日付**: 2026-07-19
- **事象**: Admonition閉じ `:::` 探索がコードフェンス内をスキップせず早期クローズした。Mermaid `run` を await/catch なしで呼び unhandled rejection / テーマ連打競合の指摘
- **原因**: 閉じ行スキャンが行全体を素朴に走査していた。クライアント描画も Promise 失敗と再入を考慮していなかった
- **ルール**: コンテナ記法の閉じ探索はフェンス（``` / ~~~）内をスキップする。非同期UI描画は suppressErrors/catch とキューで再入を防ぐ

- **日付**: 2026-07-19
- **事象**: 「サブエージェントは使用禁止」と明示されたあとでも、スキル手順どおり Bugbot サブエージェント起動を続けようとした
- **原因**: 手動アタッチされたスキル指示を、その場のユーザー上書きより優先してしまった
- **ルール**: ユーザーがサブエージェント禁止・直接実施を指示したらスキルよりユーザー指示を優先する。レビュー系も自分で差分を読んで実施する

- **日付**: 2026-07-19
- **事象**: Nunjucksマクロが返す空の SafeString を `{% if value %}` で判定すると常に truthy になり、サイドバーが全セクション展開された
- **原因**: マクロ出力は SafeString オブジェクトになり、中身が空でもオブジェクトとして truthy 扱いされる
- **ルール**: Nunjucksマクロの戻り値を真偽判定するときは `== "1"` や `| length > 0` のような明示比較を使う。空文字のまま `if` に渡さない

- **日付**: 2026-07-19

- **原因**: 安全判定が「configDir配下」だけ見て、ルート自身やdocs包含を拒否していなかった
- **ルール**: 破壊的削除のガードは「配下であること」に加え、ルート自身・ソースディレクトリ同一/包含を必ず拒否する

- **日付**: 2026-07-19

- **原因**: 「再出力」と「再変換」を同一視した。副作用フックをserve経路にも載せた。動的importのキャッシュと戻り値契約を考慮不足
- **ルール**: 増分は変更ページのみconvert。buildEndはCLIのrunBuild（mkdocsgen build）のみで、serveのfullBuild/増分ではスキップ。プラグインimportはmtimeクエリでキャッシュ破棄。transformはstring検証。decodeURIComponent失敗は400

- **日付**: 2026-07-19

- **原因**: Admonitionのフェンススキップをpydoc検出に再利用しなかった。仕様の「_始まり」を字面どおり実装しdunderを考慮しなかった。レベル上限とアンカーマージのフォールバックが不足
